# Learning Paths for Software Engineers

What to learn and in what order, based on skill demand from 1,200+ SWE job descriptions and real hiring data across FAANG, growth-stage, and Series A companies.

For a structured deep-dive with 200+ real interview examples, check out [The 0->1 SWE Interview Playbook](https://valenx.org/books/swe).


## Core Skills

The most important 20% that accounts for 80% of the work. Based on hiring manager interviews, engineering community surveys, and [job market research](../role/02-skills.md).

### Data Structures and Algorithms

- Arrays, strings, linked lists, trees, graphs, heaps, hash maps
- Sorting and searching - merge sort, binary search, BFS/DFS
- Dynamic programming - memoization, tabulation, common patterns
- Time and space complexity analysis - Big O notation, amortized analysis

Practice: solve 150-200 LeetCode problems across all categories. Focus on patterns, not memorization.

### System Design

- Distributed systems fundamentals - CAP theorem, consistency models, partitioning
- Load balancing, caching, CDNs, message queues
- Database design - SQL vs NoSQL, indexing, sharding, replication
- API design - REST, GraphQL, gRPC, versioning strategies
- Microservices architecture - service boundaries, inter-service communication
- Monitoring and observability - metrics, logging, tracing, alerting

Practice: design 10-15 systems end to end. Start simple (URL shortener, rate limiter), progress to complex (distributed file system, social media feed).

### Programming Languages and Frameworks

- Python and Java/C++ are the most in-demand combination (covering 74.3% of roles)
- JavaScript/TypeScript for full-stack and frontend positions
- Go and Rust for systems and infrastructure roles
- Framework knowledge - React, Spring Boot, Django, Express depending on stack

### Production Engineering

- CI/CD pipelines - GitHub Actions, Jenkins, GitLab CI
- Containerization - Docker, Kubernetes basics
- Cloud platforms - AWS (52.1% of roles), GCP, Azure
- Testing - unit tests, integration tests, end-to-end tests, TDD
- Version control - Git workflows, branching strategies, code review

### Software Design

- SOLID principles and clean code practices
- Design patterns - factory, observer, strategy, decorator
- Object-oriented design - class hierarchies, interfaces, composition vs inheritance
- Functional programming concepts - immutability, pure functions, higher-order functions


## Other Skills

Based on the [job market research](../role/02-skills.md). These are not covered in the core path above but frequently appear in job descriptions.

### Communication and Collaboration

- Code reviews - giving and receiving constructive feedback
- Technical documentation - design docs, API docs, runbooks
- Cross-functional communication - working with product, design, data science
- Mentoring junior engineers

### DevOps and Infrastructure

- Infrastructure as code - Terraform, CloudFormation, Pulumi
- Observability - Prometheus, Grafana, Datadog, New Relic
- Security fundamentals - authentication, authorization, encryption, OWASP
- Networking basics - HTTP/HTTPS, DNS, TCP/IP, WebSockets

### Data and ML Fundamentals

- SQL is mandatory - used in 78.3% of SWE job descriptions
- Data modeling and schema design
- Basic ML concepts - enough to work with ML teams and integrate models
- Data pipeline fundamentals - ETL, streaming vs batch

### Domain Knowledge

- Fintech - payment systems, regulatory compliance, trading platforms
- Healthcare - HIPAA, EHR systems, telehealth
- E-commerce - recommendation engines, search, inventory management
- Developer tools - SDKs, CLIs, API platforms

## Skills by Priority

### Must have

- Data structures and algorithms
- At least one strong programming language
- Git and version control
- Testing (unit, integration, E2E)
- System design fundamentals

### High value

- Cloud platforms (AWS, GCP, Azure)
- Docker and containerization
- CI/CD pipeline setup
- SQL and database design
- API design (REST, GraphQL)
- Code review and collaboration

### Differentiators

- Distributed systems expertise
- Performance optimization and profiling
- Open source contributions
- Deep domain expertise (fintech, ML infrastructure, developer tools)
- Staff/Principal level architectural thinking
