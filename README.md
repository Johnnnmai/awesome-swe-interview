### How to use the question bank

1. Start with the category most relevant to your target role
2. Work through the patterns before jumping to LeetCode
3. Practice system design with a whiteboard or drawing tool
4. Record yourself answering behavioral questions
5. Time your take-home assignments to simulate real deadlines

## 🗂️ Portfolio

Your portfolio is your proof of work. Hiring managers spend an average of 47 seconds on a portfolio before deciding whether to dig deeper.

What matters in a SWE portfolio:

- Open source contributions - shows you can work with existing codebases, follow conventions, and collaborate asynchronously
- System design documents - demonstrates architectural thinking beyond just writing code
- Side projects with real users - proves you can ship and maintain production software
- Technical blog posts - shows depth of understanding and communication skills
- Code quality in public repos - clean commits, good READMEs, tests, CI/CD

What does not matter:

- Tutorial follow-alongs (everyone has a todo app)
- Repos with a single commit from 3 years ago
- Projects without READMEs or documentation

Read the [Portfolio Strategy Guide](portfolio/README.md) for actionable steps.
